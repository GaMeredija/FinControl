# Sprint 1

## Objetivo

Fechar a base de engenharia do projeto para reduzir retrabalho nas sprints de implementacao.

## Entregaveis

- Escopo do MVP fechado
- Requisitos funcionais e nao funcionais
- Regras de negocio iniciais
- Estrutura do repositorio
- Modelagem do banco de dados
- Contrato inicial da API

## Checklist da sprint

- [x] Definir o objetivo do sistema
- [x] Priorizar o MVP
- [x] Organizar o repositorio
- [x] Documentar requisitos
- [x] Modelar entidades principais
- [x] Definir endpoints base
- [ ] Criar projeto `backend`
- [ ] Criar projeto `frontend`
- [ ] Configurar banco PostgreSQL local

## Ordem de execucao da proxima etapa

1. Implementar autenticacao
2. Implementar CRUD de contas
3. Implementar CRUD de categorias
4. Implementar CRUD de lancamentos
5. Implementar dashboard

## Definicao de pronto da Sprint 1

- Todo o time consegue entender o produto lendo a documentacao
- Existe um banco inicial pronto para implementacao
- Os principais fluxos ja estao mapeados
- O backlog do MVP esta claro o bastante para virar codigo
